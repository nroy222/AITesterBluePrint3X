# 00 — Hook · Story · Offer (Plan Before You Write)

Every piece of content is three moves: **stop them (Hook), take them somewhere (Story), ask for
something (Offer).** Get these three right on paper and the platform formats become copy work.
Get them wrong and no amount of editing saves the post.

Fill this out **once per idea.** Everything else in this chapter pulls from it.

---

## The one-line idea

> Write your idea as a single sentence. If you can't, it's not ready.

```
IDEA: ________________________________________________
WHO IT'S FOR: ________________________________________
WHY NOW: _____________________________________________  (what makes it timely / urgent)
```

---

## 1. HOOK — earn the first 3 seconds

The hook's only job is to stop the scroll and buy the next line. It is a promise, a pattern
break, or a wound you press on. It is **never** a warm-up.

### Hook patterns that work

| Pattern | Shape | Example |
|---------|-------|---------|
| **Blunt declarative** | A flat, confident claim that dares disagreement | "Your 5,000 tests are not an asset. They're a liability." |
| **Unpopular opinion** | Name the consensus, then break it | "Unpopular opinion: more coverage is making your suite worse." |
| **The reader's own thought** | Say the thing they're already half-thinking | "AI assistants get dumber after a few hours. You've noticed too." |
| **Cold-open story** | Drop them mid-scene | "A candidate walked into my interview yesterday and…" |
| **The reversal** | Set an expectation, snap it | "I automated everything. Then I deleted half of it." |
| **The number** | A specific, surprising figure (must be real) | "We cut flaky reruns 80% by changing one line." |

### Hook rules
- First line carries the whole weight. Write 5 versions, keep the sharpest.
- No stat-as-opener unless the stat *is* the shock.
- No "In this post I'll show you…" — that's a table of contents, not a hook.
- Cut every word that delays the punch.

```
HOOK (write 3–5, star the winner):
1) ___________________________________________________
2) ___________________________________________________
3) ___________________________________________________
★ WINNER: ____________________________________________
```

---

## 2. STORY — earn the trust

The middle is where you actually teach. Structure it as a small journey, not a list of facts.
Use the **Problem → Tension → Turn → Proof** spine:

- **Problem** — the real pain your audience feels (name it in their words).
- **Tension** — why the obvious fix fails / what they're getting wrong.
- **Turn** — the insight, the method, the thing you figured out.
- **Proof** — a real number, a code snippet, a before/after, a screenshot. Earn the claim.

```
PROBLEM (their pain, their words): ___________________
TENSION (why the easy answer fails): _________________
TURN (your insight / method):       _________________
PROOF (number, demo, story, code):  _________________
```

### Story rules
- One core idea per piece. Side-ideas become *future* posts, not this one.
- Specifics over adjectives. "Reduced runtime from 41s to 12s" beats "much faster."
- Make at least one line **screenshot-worthy** — a standalone, quotable truth.
- Honesty builds trust: include one real trade-off or caveat. Bias against your own interest
  where you can — it makes the rest believable.

```
SCREENSHOT LINE (the quotable one): __________________
HONEST CAVEAT (cuts against you):    _________________
```

---

## 3. OFFER — earn the next step

End with **one** clear ask. One. A post with three CTAs has zero.

### Offer ladder (pick the lightest that fits)
1. **Engagement** — a binary or specific question ("Rerun or quarantine — which one?"). Never
   "what do you think?"
2. **Save / share** — "Save this for your next sprint."
3. **Lead magnet** — comment a keyword → auto-DM a checklist / PDF / repo link.
4. **Destination** — read the full article / watch the video / open the repo.
5. **Product** — the course / tool. Use sparingly and mentor-framed, never salesy.

```
PRIMARY OFFER (exactly one): _________________________
CTA WORDING (specific, not generic): _________________
LEAD MAGNET (if any): ________________________________
```

### Offer rules
- Match the platform: LinkedIn hides body links (put links in the first comment); Instagram
  uses comment-keyword DMs; YouTube wants a one-word comment + subscribe.
- The CTA should feel like a favor to the reader, not a favor to you.

## The repurpose map

Once Hook · Story · Offer is locked, here's where each piece lands:

| Plan element | YouTube | IG Reel | IG Post | Carousel | Medium | Blog | LinkedIn |
|---|---|---|---|---|---|---|---|
| **Hook** | 0–15s hook | Slide-1 text + first spoken line | First caption line | Slide 1 | Title + bold subtitle | H1 + intro | First 2 lines |
| **Story** | Body beats | 2–4 quick beats | Caption body | Slides 2…N-1 | ## sections | ## sections | → arrow lines |
| **Proof** | On-screen demo/code | Screen-record snippet | The image | A code/stat slide | Code blocks + sources | Code + data | One stat line |
| **Offer** | One-word comment + subscribe | Comment-keyword DM | Caption CTA | Final slide CTA | Italic CTA | Inline + end CTA | First-comment link |

---

## Final gate (before you open any platform template)

- [ ] Idea fits in one sentence
- [ ] Hook stops the scroll in 3 seconds (and it's not a stat)
- [ ] Exactly one core idea
- [ ] At least one real proof point (number / code / demo)
- [ ] One screenshot-worthy line
- [ ] One honest caveat
- [ ] Exactly one primary offer
- [ ] No banned phrases

Pass all eight → go to the platform templates.
