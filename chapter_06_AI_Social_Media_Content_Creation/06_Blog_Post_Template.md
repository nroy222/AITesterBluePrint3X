# 06 — Blog Post Template (Your Own Site)

How to turn an idea into an SEO-aware blog post on **your own domain** — where you own the
traffic, the email capture, and the links (no platform reach-throttling). Pull from
[00](00_Hook_Story_Offer_Planning.md).

> A Medium article persuades. A blog post on your site does that **and** ranks. The difference is
> structure for search and intent — without sacrificing the voice.

---

## Before you write: pick the keyword & intent

```
PRIMARY KEYWORD (what they'd type into Google): ______
SEARCH INTENT (informational / how-to / comparison / decision): ______
WHO'S SEARCHING & WHAT THEY WANT NEXT: _______________
2–4 SECONDARY KEYWORDS (variations you'll use naturally): _____
```

The keyword shapes the **title, H1, URL slug, and first 100 words** — then you write like a human,
not for the crawler.

---

## Structure template

```
1. SEO title tag      (≤60 chars, keyword near the front)
2. Meta description   (≤155 chars, the promise + a reason to click)
3. URL slug           (short, keyword, hyphens)
4. # H1               (matches intent, not necessarily identical to the title tag)
5. Intro (first 100 words): hook + the keyword + the promise of what they'll get
6. ## H2 sections     (one sub-topic each; H3 for steps inside)
7. Proof: code blocks, screenshots, a table, real data
8. ## Key takeaways   (skimmable summary — also wins featured snippets)
9. ## FAQ             (3–5 real questions → schema-friendly, captures long-tail)
10. CTA               (email capture / related post / product — your funnel, your rules)
11. Internal links    (2–4 to your other posts) + 1–2 outbound to credible sources
```

## On-page SEO checklist
- [ ] Primary keyword in: title tag, H1, URL slug, first 100 words, ≥1 H2
- [ ] Title tag ≤ 60 chars, meta description ≤ 155 chars with a clear promise
- [ ] One H1 only; logical H2/H3 hierarchy
- [ ] At least one image with descriptive `alt` text
- [ ] Internal links (2–4) + outbound links to credible sources (1–2)
- [ ] Key Takeaways block (snippet bait) + FAQ (long-tail + schema)
- [ ] Readable: short paragraphs, real examples, no keyword stuffing
- [ ] Mobile-readable, fast-loading images
- [ ] No banned phrases, no fabricated numbers
